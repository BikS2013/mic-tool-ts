Plan 007: Voice Agent Command Protocol

A proposed speech-control layer for using mic-tool-ts as an oral input channel for agents while preserving normal dictation and using /end as the explicit paragraph or text-section submit marker.

Mode: design proposal Default behavior: dictation unchanged Protocol: JSON Lines No new runtime dependency required

Goal

Add a speech-control protocol that lets mic-tool-ts be used as an oral input layer for agents while preserving normal dictation.

Dictation text Continuous transcript intended for a human or plain text sink.
Text section The paragraph or section accumulated until the user says /end.
Operator state Persistent toggles such as refine, translate, and clipboard.
LLM operation Optional cleanup, translation, suggestion, criticism, or another transformation applied to a submitted section.

Recommended Approach

Implement a small state machine above finalized STT text and below output rendering. The state machine recognizes explicit spoken markers, maintains operator state, and submits the current text section when /end is spoken.

capturing_section Normal transcription. Finalized non-command text is appended to the current section buffer.
->
processing_section The submitted section is processed through active operators while new speech starts the next section.
Do not infer control intent from natural language in v1. Explicit slash-style markers are easier to test and reduce accidental processing.

Proposed Spoken Markers

Defaults should be configurable and selected for low collision with ordinary dictation.

Marker Default phrase Purpose
state_command /command <operator> <on|off> Change persistent processing state.
section_end /end Submit the current paragraph or text section for processing by active operators.
section_cancel /cancel Drop the current paragraph or text section without processing it.
literal_next literal phrase Treat the next recognized marker phrase as ordinary dictated text.

Greek equivalents can be added as aliases:

  • εντολή as an alias for /command.
  • τέλος or τέλος ενότητας as aliases for /end.
  • άκυρο as an alias for /cancel.
The existing default guard phrase, τέλος εντολής, should become one alias for section_end in protocol mode rather than staying hard-wired as a refinement-only trigger.

Stateful Operators

/command changes persistent session state. It does not by itself submit text for processing. The current section is submitted when the user says /end.

Spoken command State change
/command refine on Enable refinement for future submitted sections.
/command refine off Disable refinement.
/command translate on Enable translation for future submitted sections.
/command translate off Disable translation.
/command clipboard on Copy the final processed result of future sections to the clipboard.
/command clipboard off Disable clipboard copy.

Active pipeline at /end

raw section
  -> refine, if refine is on
  -> translate, if translate is on
  -> render / emit final output
  -> copy final output to clipboard, if clipboard is on
  • Translation runs on the complete submitted section, not partial words.
  • Greek source text translates to English under the default opposite policy.
  • English source text translates to Greek under the default opposite policy.
  • Language detection runs on the complete section after refinement when refinement is enabled, otherwise on the raw section.

Output Protocol

Agent protocol mode should emit JSON Lines. Each line is one complete UTF-8 JSON object. 

{"type":"session.started","protocol":"mic-tool-ts.voice-agent.v1","seq":1}
{"type":"state.changed","seq":2,"key":"refine","value":true}
{"type":"state.changed","seq":3,"key":"translate","value":true,"target_policy":"opposite"}
{"type":"section.submitted","seq":4,"section_id":"sec_000001","raw_text":"..."}
{"type":"section.processed","seq":5,"section_id":"sec_000001","operators":["refine","translate"],"raw_text":"...","refined_text":"...","source_language":"el","target_language":"en","output_text":"..."}
{"type":"clipboard.copied","seq":6,"section_id":"sec_000001"}
{"type":"section.cancelled","seq":7,"section_id":"sec_000002","reason":"spoken_cancel"}
{"type":"session.ended","seq":8,"reason":"SIGINT"}
  • seq is monotonically increasing per process.
  • section_id is assigned when a section starts accumulating text or when it is submitted.
  • Consumers can rely on section.processed as the event that contains the final user-facing result.
  • State changes are emitted as state.changed events.
  • Marker phrases are removed from raw_text.
  • stdout should contain either human transcript text or JSONL protocol events, not both by default.

User-Facing Modes

Mode Behavior
dictation Human-facing transcript and processed-section output on stdout.
agent-protocol JSONL state and section events on stdout. Human status and logging stay on stderr.
hybrid Plain dictation remains on stdout and protocol events go to an explicit file or pipe target.
Recommended initial implementation: ship dictation and agent-protocol first. Add hybrid only when a concrete workflow needs simultaneous human text and machine events.

Proposed Configuration

All values follow the existing four-tier resolution chain.

CLI flag Env var Default Notes
--interaction-mode <dictation|agent-protocol|hybrid> MIC_TOOL_TS_INTERACTION_MODE dictation Keeps current behavior as default.
--command-phrase <phrase> MIC_TOOL_TS_COMMAND_PHRASE /command Introduces state commands such as /command refine on.
--section-end-phrase <phrase> MIC_TOOL_TS_SECTION_END_PHRASE /end Submits the current section for processing.
--section-cancel-phrase <phrase> MIC_TOOL_TS_SECTION_CANCEL_PHRASE /cancel Discards the current section.
--literal-next-phrase <phrase> MIC_TOOL_TS_LITERAL_NEXT_PHRASE literal phrase Optional escape hatch.
--refine-default <on|off> MIC_TOOL_TS_REFINE_DEFAULT off Initial refine operator state.
--translate-default <on|off> MIC_TOOL_TS_TRANSLATE_DEFAULT off Initial translate operator state.
--translation-policy <opposite|to-en|to-el> MIC_TOOL_TS_TRANSLATION_POLICY opposite opposite: Greek to English, English to Greek.
--clipboard-default <on|off> MIC_TOOL_TS_CLIPBOARD_DEFAULT off Initial clipboard-copy state.
--protocol-output <path> MIC_TOOL_TS_PROTOCOL_OUTPUT Required for hybrid Avoids mixed stdout streams.

Implementation Sketch

Add a new protocol layer rather than expanding GuardPhraseTurnDetector until it becomes responsible for unrelated concepts.

Recommended modules

  • src/protocol/types.ts: event types, interaction modes, operator state, marker config.
  • src/protocol/markerMatcher.ts: normalized marker matching and marker stripping.
  • src/protocol/stateMachine.ts: section capture, state command parsing, /end submit, /cancel discard.
  • src/protocol/jsonlWriter.ts: line-oriented protocol sink.
  • src/protocol/controller.ts: connects renderer, refiner/translator, clipboard sink, and protocol writer.

Orchestrator wiring

  1. Resolve config.
  2. Build the base renderer.
  3. Build optional LLM refiner.
  4. Route finalized STT text through the protocol controller.
  5. If a finalized segment contains a state command, update operator state and exclude the marker from section text.
  6. If a finalized segment contains /end, submit the accumulated section through the active operator pipeline.
  7. If interactionMode === "agent-protocol", write JSONL events to stdout.
  8. Continue sending all diagnostics to stderr.

Matching Semantics

Use finalized STT text only. Partial text is too unstable and will cause false state transitions.

  • Use NFD normalization, strip combining marks, lowercase, and collapse punctuation and whitespace for ordinary phrase aliases.
  • Preserve slash-marker intent. /command, /end, and /cancel must not degrade to bare command, end, or cancel matches after punctuation normalization.
  • Match across consecutive final segments with a bounded rolling window.
  • Remove marker phrases and state commands from section payloads.
  • Prefer boundary-like matches: /command at the start of a final segment should be accepted; commandment or cancellation should not match.
  • Emit verbose diagnostics for state transitions without logging secrets.

Failure Behavior

  • If /command has an unknown operator or value, emit a warning and do not change state.
  • If /end is spoken when the current section is empty, emit a no-op diagnostic under verbose mode and do not call the LLM.
  • If refinement or translation fails after /end, emit or render the best available prior value and do not fail the process.
  • If clipboard copy fails, log under verbose and keep the processed output.
  • If shutdown occurs with a non-empty unsubmitted section, emit section.cancelled with reason shutdown before session.ended.

Testing Plan

Add focused unit tests, not live microphone tests, for the protocol logic.

  • Marker normalization matches case, accents, punctuation, and cross-final phrases while preserving slash-marker intent.
  • /command refine on|off, /command translate on|off, and /command clipboard on|off update operator state.
  • /end submits the current section and strips markers from payload.
  • /cancel drops the current section and emits section.cancelled.
  • Dictation mode remains compatible with current renderer behavior.
  • Agent protocol mode emits valid JSONL and no carriage-return overwrite artifacts.
  • Refinement success includes refined_text.
  • Translation success includes source_language, target_language, and output_text.
  • Operator failure emits the best available text only and does not exit non-zero.
  • Shutdown with an unsubmitted section emits cancellation.

Example Workflow

Plain dictation remains unchanged

mic-tool-ts

Agent protocol mode

mic-tool-ts --interaction-mode agent-protocol

Example speech

These are notes for later.
/command refine on.
/command translate on.
Open docs design project design and find the LLM refinement section.
/end.
Continue dictating normal notes.

The section before /end is processed through the active pipeline: refine first, then translate. The downstream agent consumes section.processed and ignores ambient dictation events unless it wants context.

Recommendation: build this as a protocol layer with explicit operator state, /end section submission, and JSONL events. Keep dictation as the default, make protocol events opt-in, and let active operators process complete submitted sections only.
